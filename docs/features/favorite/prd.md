# 즐겨찾기 기능 PRD

## 1. 개요

노트에 즐겨찾기를 표시해 중요한 노트를 따로 모아볼 수 있도록 한다.
즐겨찾기 상태는 `Note` 타입의 `isFavorite: boolean` 필드로 저장되며, `NoteItem` 카드에서 즉시 토글된다.
`NoteList` 상단 필터 탭으로 전체 노트와 즐겨찾기 노트를 전환할 수 있으며, 필터 탭과 검색은 독립적으로 동작한다.

---

## 2. 사용자 스토리

| #   | 역할       | 행동                                                   | 목적                                         |
| --- | ---------- | ------------------------------------------------------ | -------------------------------------------- |

| 2   | 사용자로서 | 카드에서 즐겨찾기 여부를 한눈에 확인하고 싶다          | 노트를 열어보지 않고도 상태를 파악하기 위해  |

| 4   | 사용자로서 | 즐겨찾기한 노트만 모아보고 싶다                        | 중요한 노트에 집중하기 위해                  |
| 5   | 사용자로서 | 즐겨찾기 탭에서도 검색을 사용하고 싶다                 | 즐겨찾기 안에서 특정 노트를 빠르게 찾기 위해 |

## 3. 기술 결정 (ADR)

### ADR-01. 즐겨찾기 상태 저장 위치

**Context**
즐겨찾기 상태를 어디에 저장할지 결정이 필요하다. 현재 앱은 JSON Server 기반 단일 `db.json`을 DB로 사용하며, 노트는 `/notes` 엔드포인트 하나로 관리된다. 즐겨찾기는 노트 자체의 속성이므로 노트 안에 포함시키거나 별도 컬렉션으로 분리하는 선택지가 있다.

**Decision**
`Note` 타입에 `isFavorite: boolean` 필드를 추가해 노트 문서 안에 인라인으로 저장한다.

**Alternatives**

- **별도 `/favorites` 컬렉션으로 분리**: 즐겨찾기된 노트 id 목록을 독립 엔티티로 관리 → 거부. 현재 범위에서는 과도한 복잡도다. 노트 조회 시 즐겨찾기 상태를 별도로 fetch해야 하고, JSON Server 관계 설정이 추가로 필요해진다.

**Consequences**

- (+) 노트 fetch 한 번으로 즐겨찾기 상태 포함 완전한 Note 객체 획득 가능
- (+) `updateNote`가 이미 `Partial<Note>`를 받으므로 기존 Context 구조 변경 최소
- (-) 즐겨찾기 노트만 서버에서 선택적으로 가져올 엔드포인트가 없다 — 즐겨찾기 필터링은 전체 노트를 fetch한 뒤 클라이언트에서 처리해야 한다
- (-) 태그 데이터와 마찬가지로 유효성(isFavorite 타입, 허용값)을 DB 레벨에서 강제할 수 없어 클라이언트 검증에만 의존

### ADR-02. 토글 저장 타이밍

**Context**
즐겨찾기 토글을 언제 서버에 반영할지 결정이 필요하다. 태그·제목·내용은 편집 후 저장 버튼 클릭 시 일괄 저장되지만, 즐겨찾기는 성격이 다르다. 노트 내용 수정이 아닌 상태 표시이므로 사용자가 저장 버튼을 누를 의도 없이 카드에서 빠르게 토글하는 사용 패턴이 자연스럽다.

**Decision**
별 아이콘 클릭 즉시 PATCH 요청을 보낸다. 저장 버튼과 무관하게 동작한다.

**Alternatives**

- **저장 버튼 클릭 시 일괄 저장**: 태그·제목·내용과 동일한 저장 흐름 → 거부. NoteItem 카드에서 토글 후 저장 버튼을 찾아 눌러야 하는 흐름이 부자연스럽다. NoteEditor가 열려 있지 않은 상태에서는 저장 버튼 자체가 없다.

**Consequences**

- (+) 카드 목록에서 NoteEditor를 열지 않고도 즐겨찾기 상태를 변경 가능
- (+) 사용자 의도와 저장 타이밍이 일치해 UX가 직관적
- (-) 낙관적 업데이트 없이 서버 응답 후 갱신하므로, 느린 네트워크에서 별 아이콘이 반응 전까지 지연될 수 있음
- (-) 토글마다 개별 PATCH 요청이 발생해 빠르게 반복 토글 시 요청이 누적될 수 있음 → 버튼 `disabled`로 중복 클릭 방지

### ADR-03. 필터 탭 상태 관리 위치

**Context**
"전체 / 즐겨찾기" 필터 탭의 선택 상태(`filterMode`)를 어디서 관리할지 결정이 필요하다. 현재 `searchQuery`는 `App` 로컬 state로 관리되고 `NoteList`에 prop으로 전달된다. `filterMode`도 성격이 유사하다.

**Decision**
`App` 로컬 state로 `filterMode: 'all' | 'favorite'`를 관리한다. `searchQuery`와 동일한 패턴으로 `NoteList`에 prop으로 전달한다.

**Alternatives**

- **Context에서 filterMode 관리**: 전역 상태로 어디서든 접근 가능 → 거부. filterMode는 UI 흐름 상태로 서버 데이터와 무관하다. 현재 Context의 역할(서버 데이터·CRUD 액션)을 벗어나며, `searchQuery`와 일관성도 깨진다.
- **NoteList 로컬 state**: NoteList 안에서만 관리 → 거부. 탭 전환 시 `selectedNoteId` 초기화가 필요한데, 이 상태는 App이 소유하므로 NoteList에서 접근할 수 없다.

**Consequences**

- (+) `searchQuery`와 동일한 패턴 — 기존 구조와 일관성 유지
- (+) 탭 전환 시 `selectedNoteId` 초기화를 App에서 함께 처리 가능
- (-) filterMode가 App에 고정되므로, 이후 다른 위치(예: 헤더의 즐겨찾기 수 표시)에서 접근하려면 prop drilling이 다시 발생한다 — 접근 범위가 넓어지면 Context로 올려야 하는 시점이 온다

---

### ADR-04. 토글 시 updatedAt 갱신 여부

**Context**
현재 `api.updateNote`는 PATCH 요청 body에 항상 `updatedAt: new Date().toISOString()`을 포함한다. 즐겨찾기 토글도 내부적으로 PATCH이므로 기존 함수를 재사용하면 `updatedAt`이 갱신된다. 즐겨찾기는 노트 내용 수정이 아니므로 `updatedAt` 갱신이 적절한지 판단이 필요하다.

**Decision**
토글 시 `updatedAt`을 갱신하지 않는다. `api.ts`에 `toggleFavorite(id, isFavorite)` 전용 함수를 추가해 `isFavorite` 필드만 PATCH한다.

**Alternatives**

- **기존 `api.updateNote` 재사용**: 별도 함수 없이 `updateNote(id, { isFavorite })` 호출 → 거부. `updateNote`는 항상 `updatedAt`을 갱신하므로 즐겨찾기 토글이 노트 수정 시간에 영향을 준다. 노트 목록의 "최근 수정" 순서가 즐겨찾기 동작으로 바뀌는 부작용이 생긴다.

**Consequences**

- (+) 즐겨찾기 토글이 노트의 수정 시간(`updatedAt`)에 영향을 주지 않음
- (+) 토글 의도와 데이터 의미가 명확하게 분리됨
- (-) `api.ts`에 함수가 하나 추가됨
- (-) Context 액션도 `toggleFavorite`을 별도로 추가해야 해 기존 `updateNote`와 역할이 분리됨

## 4. Out of Scope

- 즐겨찾기 노트 상단 고정 (정렬 순서 변경)
- 즐겨찾기 수 배지 표시 (탭에 숫자 표기)
- 즐겨찾기 일괄 해제
- 즐겨찾기 기반 알림·리마인더
- 즐겨찾기 탭 내 별도 정렬 옵션

## 5. 용어 정의

| 용어                | 정의                                                                                |
| ------------------- | ----------------------------------------------------------------------------------- |
| 즐겨찾기(favorite)  | 노트에 붙이는 중요도 표시. `isFavorite: boolean` 값으로 저장                        |
| 토글(toggle)        | 즐겨찾기 상태를 true ↔ false로 전환하는 동작. 별 아이콘 클릭으로 실행               |
| 필터 탭(filter tab) | NoteList 상단의 "전체 / 즐겨찾기" 전환 UI                                           |
| filterMode          | App이 소유하는 필터 상태값. `'all'`이면 전체, `'favorite'`이면 즐겨찾기 노트만 표시 |
| 즉시 PATCH          | 저장 버튼과 무관하게 토글 시점에 바로 서버에 PATCH 요청을 보내는 저장 방식          |
