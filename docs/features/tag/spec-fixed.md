# 태그 기능 상세 스펙

## 결정 요약

| 항목        | 결정                                                          |
| ----------- | ------------------------------------------------------------- |
| 저장 방식   | `Note` 타입에 `tags: string[]` 필드 추가, 노트 내 배열로 저장 |
| 입력 위치   | `NoteEditor` 내부 (제목/내용 폼 하단)                         |
| 입력 UX     | 텍스트 인풋 + Enter/쉼표로 태그 추가, 칩(badge) 형태 표시     |
| 유효성 검사 | 최대 15자, 한글/영문/숫자/하이픈(`-`)만 허용                  |
| 태그 필터링 | 이번 구현 범위 제외                                           |

---

## 데이터 구조

### Note 타입 변경

```ts
// src/types/note.ts
interface Note {
  id: string;
  title: string;
  content: string;
  tags: string[]; // 추가
  createdAt: string;
  updatedAt: string;
}
```

### 저장 규칙

- 태그는 입력 시 **소문자로 정규화**해서 저장 (`React` → `react`)
- 중복 검사는 정규화 이후 기준으로 판단
- 노트당 최대 **10개**까지 허용
- 기존 노트에 `tags` 필드가 없을 경우 `[]`로 기본값 처리

---

## 유효성 검사 규칙

| 규칙      | 내용                                                                 |
| --------- | -------------------------------------------------------------------- |
| 빈 값     | 공백만 있는 입력 무시                                                |
| 최대 길이 | 15자 초과 시 추가 거부                                               |
| 허용 문자 | 한글, 영문, 숫자, 하이픈(`-`)만 허용 (정규식: `/^[a-z0-9가-힣-]+$/`) |
| 중복      | 동일 태그(소문자 기준) 추가 시 무시                                  |
| 최대 개수 | 10개 초과 시 추가 거부                                               |

## UI 동작

### 태그 입력

1. `NoteEditor` 폼 하단에 태그 입력 인풋 추가
2. 텍스트 입력 후 `Enter` 또는 `,` 입력 시 태그 추가
3. 유효성 검사 통과 시 칩으로 변환, 입력창 초기화
4. 유효성 검사 실패 시 입력창을 흔들리는 애니메이션으로 피드백 (toast/alert 없이)

### 태그 표시

- 칩(badge) 형태로 인풋 위에 나열

### 태그 삭제

- 칩에 마우스를 올리면(hover) `X` 버튼이 나타남 (평상시에는 숨김)
- `X` 버튼 클릭 시 해당 태그 제거
- 로컬 상태만 즉시 반영, 서버 반영은 저장 버튼 클릭 시

### 저장

- 기존 저장 버튼 클릭 시 제목/내용/태그 한 번에 `updateNote` (PATCH)
- 낙관적 업데이트 없음, 서버 응답 후 Context 상태 갱신

### NoteItem 카드

- 태그는 카드 목록에서 표시하지 않음 (NoteEditor에서만 표시)

## 엣지 케이스

### 중복 처리

| 상황                             | 처리                                                      |
| -------------------------------- | --------------------------------------------------------- |
| 이미 존재하는 태그 입력          | 소문자 정규화 후 비교, 동일하면 추가 무시 + 흔들림 피드백 |
| `React`와 `react` 동시 입력 시도 | 정규화 기준 동일 태그로 판단 → 두 번째 입력 무시          |

### 공백 처리

| 상황                     | 처리                                      |
| ------------------------ | ----------------------------------------- |
| 앞뒤 공백 (`" react "`)  | trim 후 유효성 검사 진행                  |
| 중간 공백 (`"react js"`) | 허용 문자 정규식에서 거부 → 흔들림 피드백 |
| 공백만 입력 (`"   "`)    | trim 결과 빈 문자열 → 추가 무시           |
| 입력 없이 Enter          | 아무 동작 없음                            |

### 저장 형태

| 상황                     | 처리                                                      |
| ------------------------ | --------------------------------------------------------- |
| 태그 저장 시             | 소문자 정규화된 문자열 배열로 저장 (`["react", "study"]`) |
| 저장 없이 다른 노트 클릭 | 태그 변경사항 버림 (제목/내용과 동일한 동작)              |
| 새 노트 생성 시          | `tags: []` 빈 배열로 초기화하여 POST                      |

### 최대 길이

| 상황           | 처리                                                  |
| -------------- | ----------------------------------------------------- |
| 15자 초과 입력 | 인풋에서 16번째 글자부터 입력 차단 (`maxLength={15}`) |
| 정확히 15자    | 허용                                                  |

### 최대 개수

| 상황                         | 처리                                                                                 |
| ---------------------------- | ------------------------------------------------------------------------------------ |
| 태그 10개 상태에서 추가 시도 | 인풋 비활성화(`disabled`) + "태그는 최대 10개까지 추가할 수 있습니다" 안내 문구 표시 |
| 태그 삭제 후                 | 인풋 다시 활성화                                                                     |

### 빈 노트

| 상황                                   | 처리                             |
| -------------------------------------- | -------------------------------- |
| `tags` 필드 없는 기존 노트             | `note.tags ?? []` 로 기본값 처리 |
| 노트 선택 전 (에디터 미표시 상태)      | 태그 입력 UI 렌더링 안 함        |
| 새 노트 작성 중 태그 입력 후 저장 취소 | 태그 포함 전체 상태 버림         |
