# 태그 필터링 요구사항

## 개요

NoteItem 카드에 태그 칩을 표시하고, 클릭 시 해당 태그가 붙은 노트만 목록에 표시한다.
태그 칩을 다시 클릭하면 필터가 해제된다. 서버 변경 없이 클라이언트 사이드 필터링으로 구현한다.

---

## 사용자 시나리오

**시나리오 1 — 태그로 노트 필터링**

노트 카드에 표시된 태그 칩을 클릭하면 해당 태그를 가진 노트만 목록에 표시된다.
사이드바 상단에 "#{태그명} 필터 적용 중" 텍스트가 표시된다.

**시나리오 2 — 필터 해제**

활성화된 태그 칩을 다시 클릭하면 필터가 해제되고 전체 노트 목록이 복귀한다.

**시나리오 3 — 필터 교체**

필터가 걸린 상태에서 다른 태그 칩을 클릭하면 새 태그로 교체된다.
이전 필터는 자동으로 해제된다.

**시나리오 4 — 검색과 동시 적용**

텍스트 검색어와 태그 필터가 동시에 적용되면 AND 조건으로 동작한다.
두 조건을 모두 만족하는 노트만 표시된다.

---

## 경계 조건 및 엣지 케이스

- 태그 필터 적용 결과가 0건이면 기존 "검색 결과가 없습니다" 빈 상태 메시지 재사용
- 태그가 없는 노트는 태그 칩 영역을 렌더링하지 않음
- 한 번에 하나의 태그만 필터링 가능 (단일 선택)

---

## 에러 처리 방식

클라이언트 사이드 필터링이므로 별도 에러 처리 없음. API 호출 없이 기존 notes 배열을 필터링한다.

---

## Out of Scope (초안)

- 다중 태그 동시 필터링 (AND/OR 복수 선택)
- 사이드바에 전체 태그 목록 패널 표시
- 태그 기반 검색 (검색창에 태그 입력)
- NoteItem 카드 태그 수정

---

## 용어 정의

| 용어                      | 정의                                                                  |
| ------------------------- | --------------------------------------------------------------------- |
| **태그 필터(tag filter)** | 특정 태그를 기준으로 노트 목록을 좁히는 클라이언트 사이드 필터링 동작 |
| **활성 태그(active tag)** | 현재 필터링에 사용 중인 태그. `null`이면 필터 없음                    |
| **태그 칩(tag chip)**     | NoteItem 카드 하단에 표시되는 클릭 가능한 태그 UI 요소                |
| **토글(toggle)**          | 같은 태그 칩을 재클릭해 활성 태그를 켜고 끄는 동작                    |
