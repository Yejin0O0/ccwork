# Spacing

## 기본 단위

**1.4rem** 을 기준 리듬으로 사용한다.

---

## 간격 스케일

| 토큰         | 값      | 용도                         |
| ------------ | ------- | ---------------------------- |
| `spacing.1`  | 0.35rem | 레이블 → 인풋 간격           |
| `spacing.2`  | 0.7rem  | Headline → Body 간격         |
| `spacing.4`  | 1.4rem  | 리스트 항목 사이 간격 (표준) |
| `spacing.6`  | 2.1rem  | 컴포넌트 간 간격             |
| `spacing.10` | 3.5rem  | 레이아웃 섹션 간 간격        |

---

## 수직 리듬

```
[Display / Headline]
    ↓ spacing.2 (0.7rem)
[Body Text]
    ↓ spacing.4 (1.4rem)
[다음 항목 / 컴포넌트]
    ↓ spacing.10 (3.5rem)
[다음 섹션]
```

---

## 레이아웃 섹션 간격

대형 레이아웃 블록 사이는 `spacing.10` (3.5rem)을 사용한다.
"Atmospheric" 품질을 유지하기 위해 충분한 여백이 필요하다.

```
헤더 영역
    ↓ 3.5rem
노트 목록 섹션
    ↓ 3.5rem
푸터 / 하단 액션
```

---

## 여백 원칙

> 여백을 구조적 요소로 사용한다. 확신이 없으면 마진을 늘린다.

- 콘텐츠가 숨막히게 붙어 있으면 `spacing.4` 이상으로 늘린다
- 리스트 구분선 대신 `spacing.4` 간격으로 항목을 분리한다
- 섹션 타이틀과 본문 사이는 `spacing.2` 이상을 확보한다

---

## Do / Don't

### ✅ Do

- 리스트 항목 간격은 `spacing.4` (1.4rem)를 기본으로 사용한다
- Headline과 Body 사이는 `spacing.2` (0.7rem)을 사용한다
- 레이아웃 섹션 사이는 `spacing.10` (3.5rem)으로 여유를 준다

### ❌ Don't

- 간격을 8px 단위 시스템(8, 16, 24px)으로 임의 설정하지 않는다 — 1.4rem 기반 사용
- 리스트 항목 구분을 위해 `border-bottom`을 사용하지 않는다 — 간격으로 대체
- 섹션 간격을 `spacing.4` 이하로 좁히지 않는다 — Atmospheric 품질 유지
